# Code of Conduct

## Purpose

LexiconAI is intended to be a professional, respectful, and inclusive project.
This Code of Conduct defines the standards expected in issues, pull requests,
code review, discussions, and any other project spaces.

## Expected Behavior

Contributors and maintainers are expected to:

- Communicate respectfully and assume good intent until there is evidence otherwise.
- Give technical feedback that is specific, constructive, and relevant to the work.
- Be open to correction, discussion, and reasonable disagreement.
- Keep discussions focused on improving the project and helping others contribute.
- Respect differences in experience level, background, identity, and perspective.

## Unacceptable Behavior

The following behavior is not acceptable in project spaces:

- Harassment, intimidation, threats, or personal attacks.
- Discriminatory or derogatory language directed at any individual or group.
- Trolling, deliberate provocation, or sustained disruptive behavior.
- Publishing or threatening to publish private, personal, or identifying information.
- Sexualized language or imagery in discussions, issues, pull requests, or reviews.
- Repeatedly derailing technical discussions after moderation or maintainer guidance.

## Scope

This Code of Conduct applies to all project-managed spaces, including:

- GitHub issues, discussions, pull requests, and code reviews.
- Documentation feedback and support conversations related to the project.
- Any public communication clearly representing the project or its community.

## Reporting Concerns

If you experience or witness behavior that violates this Code of Conduct, report it
to the project maintainer privately through GitHub or another trusted contact path
made available by the repository owner.

When possible, include:

- A description of what happened.
- Links to the relevant discussion, issue, pull request, or comment.
- Any context that would help with a fair review.

Reports should be made in good faith. The maintainer will review available
information, protect confidentiality where practical, and take action that is
proportionate to the circumstances.

## Maintainer Responsibilities

Project maintainers may remove, edit, or moderate comments, commits, code review
feedback, issues, or other contributions that violate this Code of Conduct.

Maintainers are also responsible for:

- Applying standards consistently and in good faith.
- Explaining moderation decisions when appropriate.
- Prioritizing the safety and usability of project spaces.

## Enforcement

Responses to violations may include:

- A private warning.
- Removal of content.
- Temporary or permanent restrictions from participation.
- Escalation to platform-level reporting where appropriate.

Deliberate or repeated violations may result in a permanent ban from project
participation.

## Project Commitment

Healthy collaboration improves the quality of the software and the sustainability
of the project. Everyone participating in LexiconAI is expected to help maintain a
respectful working environment.
